The error occurs because the GitHub Actions environment doesn't have Chrome/Chromium installed in the expected location. Here's how to fix it:

Add Chrome Installation to Your Workflow
Update your GitHub Actions workflow (.github/workflows/your-workflow.yml) to install Chrome:

yaml
jobs:
  your-job:
    runs-on: ubuntu-latest
    steps:
      # ... other steps ...
      
      - name: Install Chrome
        run: |
          sudo apt-get update
          sudo apt-get install -y chromium-browser
          
      - name: Run Tests
        run: |
          export TEST_URL=http://localhost:5000
          npm test  # or your test command
Configure Selenium to Use the Correct Binary Path
In your test setup, specify the Chrome binary location:

javascript
const { Builder } = require('selenium-webdriver');
const chrome = require('selenium-webdriver/chrome');

async function setupDriver() {
  const options = new chrome.Options();
  options.setChromeBinaryPath('/usr/bin/chromium-browser');
  options.addArguments('--headless=new');
  options.addArguments('--no-sandbox');
  options.addArguments('--disable-dev-shm-usage');

  return await new Builder()
    .forBrowser('chrome')
    .setChromeOptions(options)
    .build();
}

Install chromedriver package and use WebDriver Manager:

npm install chromedriver

Full Working GitHub Action Example

yaml
name: CI
on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    
    - name: Setup Node
      uses: actions/setup-node@v4
      with:
        node-version: 18
        
    - name: Install dependencies
      run: npm ci
      
    - name: Install Chrome
      run: |
        sudo apt-get update
        sudo apt-get install -y chromium-browser
        
    - name: Run Tests
      run: |
        export TEST_URL=http://localhost:5000
        npm test
Key fixes:

Explicit Chrome installation via apt-get

Correct binary path configuration (/usr/bin/chromium-browser)

Added necessary Chrome arguments for headless mode

Ensure Chrome and ChromeDriver version compatibility

If you're using ChromeDriver directly, consider adding the ChromeDriver setup action:

yaml
- name: Set up Chrome Driver
  uses: nanasess/setup-chromedriver@v1
  with:
    chromedriver-version: 'latest'
Then update your test command to start ChromeDriver first:

yaml
- name: Run Tests
  run: |
    export TEST_URL=http://localhost:5000
    chromedriver --url-base=/wd/hub &
    npm test
